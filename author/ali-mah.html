Hot News Come From tweets¶

Say we want to separate the really hot news from the bulk of regular updates. One method is to look at the tweets from news agencies. But there are still normal important feeds all mixed up together. In this the following two posts, I presnt you my apploach to extract hot topics from the news feeds.

Extract and Store Data¶

Any data driven analysis requires valid data. Here, we use tweeter to extract post from news agengies. The data is stored locally in a MongoDB database and is later used to gain ingight regarding the trends and hot news. The plan is to run this code frequently (like every 5 min) and monitor the new tweets in each retreival. Therfore, the id and timestamp of last captured tweet is stored for each agency and used in the next iteration for the starting point.

the tweet_search() method is our key tweet capturing function. For each given target user (news agency in our case), it returns all the new tweets. In this work, after each such capturing, we explicitally store the id of the last tweet (max_id) and used it as the starting point of the next round (since_id).

The tweet capturing procedure is: For each news agency:

1. check the data base for the *id* of the last stored tweet
    1.1. if such history exists: use that for the starting point of reading new tweets
    1.2. if not: generate a starting point based on the time you want between readings
2.use gather_news() method to read, clean and store new tweets

Now that we decided on the starting pointm we can read new tweets. Then we parse the tweet json and store useful information in the database. We additionally store the timestamp and id of the last tweet for later recalls.

def gather_tweets(api,screen_name, max_tweets, since_id, time_limit,bulk,read_since_IDs):
    '''going through tweets'''
    start = dt.datetime.now()
    end = start + dt.timedelta(seconds=time_limit)
    count, exitcount = 0, 0
    max_timestamp = -1

    while dt.datetime.now() < end: 
        count += 1
        if debug:
            print('count =',count)
        
        # collect tweets and update max_id
        tweets, max_id = tweet_search(api, screen_name, max_tweets,
                                       since_id=since_id)
        since_id = max_id
        if tweets:
            for tweet in tweets:
                
                # prepare tweet features to be stored in mongodb collection
                [tweet_ctime, tweet_id, text, hashtags, users, urls, media_urls, nfollowers, nfriends] = parse_json_tweet(tweet)
                # igonre tweets with no created_at time
                if tweet_ctime:
                    # provide a timestamp for each tweet from its created_at time
                    tweet_mytime = int(time.mktime(tweet_ctime.timetuple()))
                    
                    # get the maximum timestamp to be kept as an information in our collectionNA
                    if tweet_mytime > max_timestamp:
                        max_timestamp = tweet_mytime

                    Item={'source':screen_name,'id':tweet_id,
                        'time_in_secs':tweet_mytime,
                        'text':text,'created_at':tweet_ctime,
                        'hashtags':hashtags,'users':users, 'media_urls':media_urls, 'urls':urls,
                        'nfollowers':nfollowers,'nfriends':nfriends}
                    bulk.insert(Item)    
                                                
            exitcount = 0
            bulk.execute()
            
            if (not read_since_IDs):
                break
        # if for exitcount=3 times, no tweet found, we'll break the loop
        else:
            exitcount += 1
            if exitcount == setting.EXITCOUNT:
                # if screen_name == search_phrases[-1]:
                # 	sys.exit('Maximum number of empty tweet strings reached - exiting')
                # else:
                if debug:
                    print('Maximum number of empty tweet strings reached - break')
                break
    

    if since_id != -1:
        key = {'news_agencies':screen_name}
        collectionNA.update({'news_agencies':screen_name}, {"$set":{'max_id':since_id, 'max_timestamp': max_timestamp, 'modified_date':dt.datetime.now()}}, upsert = True)

'''convert UTC time to the local time'''
def utcToLocal(utcTime):
        #Auto-detect zones:
        from_zone = tz.tzutc()
        to_zone = tz.tzlocal()
        # Tell the datetime object that it's in UTC time zone since 
        # datetime objects are 'naive' by default
        utc = utcTime.replace(tzinfo = from_zone)
        # Convert time zone
        return utc.astimezone(to_zone)

# format a UTC time converted to the local time
def formatUtcToLocal(utcTime, format='%d-%m-%Y %H:%M:%S'):
    return utcToLocal(utcTime).strftime(format)

# get features of each tweet from the twitter API which is in JSON format
def parse_json_tweet(tweet):
    # skip non-enlish tweets
    if tweet.lang != 'en':
        return [None, None , '', [], [], [],[],0,0]

    date = tweet.created_at
    id = tweet.id
    nfollowers = tweet.user.followers_count
    nfriends = tweet.user.friends_count

    text = tweet.text

    hashtags=[]
    hashtags = [hashtag.get('text',[]) for hashtag in tweet.entities.get('hashtags',{})]

    users=[]
    users = [user_mention.get('screen_name',[]) for user_mention in tweet.entities.get('user_mentions',{})]
    
    urls = []
    urls = [url.get('expanded_url',[]) for url in tweet.entities.get('urls',{})]
    
    media_urls = []
    media_urls = [media.get('media_url',[]) for media in tweet.entities.get('media',{})]  

    return [date, id, text, hashtags, users, urls, media_urls, nfollowers, nfriends]
